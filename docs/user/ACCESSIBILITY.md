# Accessibility Guide

Complete guide to accessibility features in the Disability Advocacy app.

## Overview

The Disability Advocacy app is designed with accessibility in mind. We've implemented features to make the app usable by everyone, regardless of their abilities.

## VoiceOver Support

### iOS VoiceOver

The app is fully compatible with iOS VoiceOver:

- **Navigation**: All UI elements have proper accessibility labels
- **Content**: Text content is read aloud clearly
- **Actions**: All buttons and interactive elements are accessible
- **Lists**: Lists are navigable with VoiceOver gestures
- **Forms**: Forms are accessible and easy to navigate

### macOS VoiceOver

Full VoiceOver support on macOS:

- **Keyboard Navigation**: Navigate the entire app with keyboard
- **Screen Reader**: All content is accessible to screen readers
- **Menu Items**: All menu items are properly labeled
- **Dialogs**: System dialogs are accessible

### Using VoiceOver

**iOS:**
1. Go to Settings > Accessibility > VoiceOver
2. Enable VoiceOver
3. Use gestures to navigate:
   - Swipe right to move forward
   - Swipe left to move back
   - Double-tap to activate

**macOS:**
1. Go to System Settings > Accessibility > VoiceOver
2. Enable VoiceOver (or press Command + F5)
3. Use keyboard shortcuts to navigate

## Dynamic Type Support

### Adjusting Text Size

The app supports Dynamic Type, allowing you to adjust text size system-wide:

**iOS:**
1. Go to Settings > Display & Brightness > Text Size
2. Adjust the slider to your preferred size
3. The app will automatically adapt

**macOS:**
1. Go to System Settings > Accessibility > Display
2. Adjust text size settings
3. The app will adapt to your preferences

### Text Size Options

- **Small**: Default text size
- **Medium**: Slightly larger
- **Large**: Larger text for better readability
- **Extra Large**: Maximum text size
- **Accessibility Sizes**: Extra large sizes for accessibility needs

## High Contrast Mode

### Enabling High Contrast

**In App:**
1. Go to Settings > Accessibility Settings
2. Enable "High Contrast Mode"
3. The app will use higher contrast colors

**System-Wide (macOS):**
1. Go to System Settings > Accessibility > Display
2. Enable "Increase Contrast"
3. The app will respect system settings

### Benefits

- Better visibility for low vision users
- Reduced eye strain
- Clearer distinction between UI elements

## Reduced Motion

### Enabling Reduced Motion

**iOS:**
1. Go to Settings > Accessibility > Motion
2. Enable "Reduce Motion"
3. The app will reduce animations

**macOS:**
1. Go to System Settings > Accessibility > Display
2. Enable "Reduce Motion"
3. Animations will be minimized

### Benefits

- Reduced motion sickness
- Better focus on content
- Faster navigation for some users

## Keyboard Navigation

### Full Keyboard Support

The app supports complete keyboard navigation:

**Navigation:**
- Tab: Move forward through elements
- Shift+Tab: Move backward
- Enter/Space: Activate buttons
- Arrow keys: Navigate lists and menus
- Escape: Close dialogs or go back

**Shortcuts (macOS):**
- Command+F: Search
- Command+,: Settings
- Command+W: Close window
- Command+Q: Quit app

### Customizable Shortcuts

On macOS, you can customize keyboard shortcuts:
1. Go to Settings > Keyboard Shortcuts
2. View available shortcuts
3. Customize as needed

## Color and Contrast

### Color Accessibility

- **High Contrast**: All text meets WCAG contrast requirements
- **Color Independence**: Information isn't conveyed by color alone
- **Dark Mode**: Full support for Dark Mode
- **Customizable**: Adjust colors in Accessibility Settings

### Color Blind Support

- **Patterns**: Icons and patterns supplement color coding
- **Labels**: All color-coded information has text labels
- **High Contrast**: High contrast mode improves visibility

## Screen Reader Optimization

### Proper Labeling

All UI elements have:
- **Accessibility Labels**: Descriptive labels for screen readers
- **Hints**: Helpful hints for complex interactions
- **Values**: Current values are announced
- **States**: Button states (selected, disabled) are announced

### Content Structure

- **Headings**: Proper heading hierarchy
- **Lists**: Properly structured lists
- **Forms**: Labeled form fields
- **Landmarks**: Clear navigation landmarks

- **iOS**: At least 44x44 points

### Spacing

Adequate spacing between interactive elements to prevent accidental taps.

## Testing Accessibility

### How We Test

- **VoiceOver Testing**: Regular testing with VoiceOver enabled
- **Keyboard Testing**: Full keyboard navigation testing
- **Contrast Testing**: Automated contrast ratio checking
- **User Testing**: Testing with users who have disabilities

### Reporting Issues

If you encounter accessibility issues:

1. **Document the issue**: Note what feature isn't accessible
2. **Include details**: Device, OS version, app version
3. **Report**: Use the app's feedback system
4. **We'll fix it**: Accessibility issues are high priority

## Tips for Best Experience

1. **Customize Settings**: Adjust accessibility settings to your needs

3. **Adjust Text Size**: Use Dynamic Type for comfortable reading
4. **Enable High Contrast**: If you have low vision, high contrast helps
5. **Keyboard Navigation**: Learn keyboard shortcuts for faster navigation
